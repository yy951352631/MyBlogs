SpringCloud配置中心

1、问题：SpringCloud配置中心为何物？

1.1 背景&概念：

微服务意味着要将单体应用中的业务拆分成一个个子服务，每个服务的粒度相对较小，因此系统中会出现大量的服务。由于每个服务都需要必要的配置信息才能运行，所以一套集中式的、动态的配置管理设施是必不可少的。SpringCloud提供了ConfigServer来解决这个问题，我们每一个微服务自己带着一个application.yml，上百个配置文件的管理，使得我们分身乏术，运维工程师心态爆炸。因而SpringCloud Config为微服务架构中的微服务提供集中化的外部配置支持，配置服务器为各个不同微服务应用的所有环境提供了一个中心化的外部配置。

1.2 原理简介：

我们的配置服务中心主要由三部分组成，即我们的【git仓库】—>【我们的配置服务中心】—>【我们的客户端】这里的客户端（client）是指需要从配置中心获取配置信息的微服务模块。其工作流程简单来说便是

1）由我们的【配置服务中心】实时的读取git上的信息变化

2）我们需要远程配置的客户端再通过【配置服务中心】获取到git上最新的配置信息并应用到本身。

3）由于微服务模块很可能也会有自己的配置文件，这就引出了一个问题—如何保证我们的远端配置信息能够顺利的覆盖本地配置信息，从而进行有效的修改？这就引出了一个新的配置文件：bootstrap.yml。

1、问题：profile是个啥？

1.1 profile干嘛的？

Profile 可以让 Spring 对不同的环境提供不同配置的功能，可以通过激活、指定参数等方式快速切换环境。

1.2 profile的两种使用形式

1）通过创建多个配置文件的方式使用profile

1.1）创建文件如下

  • application.properties：主配置文件
  • application-dev.properties：开发环境配置文件
  • application-test.properties：测试环境配置文件
  • application.prop-properties：生产环境配置文件
