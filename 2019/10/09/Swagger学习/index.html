Swagger

1、背景

前后端集成联调，前端人员和后端人员无法做到：“及时协商，尽早解决”，最终导致问题集中爆发

2、解决方案

1）首先制定schema【计划提纲】，实时更新最新的API，降低集成的风险

2）早些年：制定word文档

3）前后端分离：

​ 1.前端测试后端接口：postman

​ 2.后端提供接口，需要实时更新最新的消息及改动

3、引出Swagger

1）号称世界上最流行的API框架

2）RestFul Api 文档在线自动生成工具—>Api文档与Api定义同步更新

3）直接运行，可以在线测试Api接口

4）支持多种语言
